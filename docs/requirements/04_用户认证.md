# 用户认证功能需求与方案

## 1. 需求概述

| 项目 | 内容 |
|------|------|
| 功能名称 | 用户认证系统 |
| 功能描述 | 实现用户注册、登录、登出功能，支持 JWT Token 认证 |
| 优先级 | P0（核心功能） |

## 2. 功能需求

### 2.1 用户注册

- 支持用户名 + 密码注册
- 密码需加密存储（bcrypt）
- 用户名唯一性校验

### 2.2 用户登录

- 用户名 + 密码登录
- 登录成功后返回 JWT Token
- Token 有效期 7 天

### 2.3 用户登出

- 前端清除 Token 即可

### 2.4 路由保护

- 需要认证的接口通过 JWT Token 验证
- Token 无效或过期返回 401

### 3.4 核心代码实现

#### 依赖注入（获取当前用户）

```python
from fastapi import Depends, HTTPException, status
from fastapi.security import HTTPBearer, HTTPAuthorizationCredentials
import jwt

security = HTTPBearer()

def get_current_user(credentials: HTTPAuthorizationCredentials = Depends(security)):
    token = credentials.credentials
    try:
        payload = jwt.decode(
            token, 
            settings.SECRET_KEY, 
            algorithms=[settings.ALGORITHM]
        )
        user_id = payload.get("sub")
        if user_id is None:
            raise HTTPException(
                status_code=status.HTTP_401_UNAUTHORIZED,
                detail="无效的 Token"
            )
        return {"user_id": user_id}
    except jwt.ExpiredSignatureError:
        raise HTTPException(
            status_code=status.HTTP_401_UNAUTHORIZED,
            detail="Token 已过期"
        )
    except jwt.InvalidTokenError:
        raise HTTPException(
            status_code=status.HTTP_401_UNAUTHORIZED,
            detail="无效的 Token"
        )
```

#### 受保护的路由示例

```python
@router.get("/me")
def get_me(current_user: dict = Depends(get_current_user)):
    return {"user_id": current_user["user_id"]}
```

## 4. 前端对接

### 4.1 Token 存储

```javascript
// 登录成功后保存 Token
localStorage.setItem('token', response.access_token)

// 请求时携带 Token
const response = await fetch('/api/chat', {
    headers: {
        'Authorization': `Bearer ${localStorage.getItem('token')}`
    }
})
```

### 4.2 登录状态判断

```javascript
// 检查是否已登录
const isLoggedIn = () => {
    return !!localStorage.getItem('token')
}

// 登录页面判断
if (isLoggedIn()) {
    router.push('/chat')
}
```

### 4.3 路由守卫

**作用**：未登录用户访问需要认证的页面时，自动跳转到登录页。

#### 路由配置

```typescript
// router/index.ts
const routes = [
    {
        path: '/login',
        name: 'Login',
        component: () => import('@/views/Login.vue')
    },
    {
        path: '/chat',
        name: 'Chat',
        component: () => import('@/views/Chat.vue'),
        meta: { requiresAuth: true }  // 需要登录
    },
    // ... 其他需要登录的页面
]
```

#### 守卫实现

```typescript
// router/index.ts
import router from './index'

router.beforeEach((to, from, next) => {
    const token = localStorage.getItem('token')
    
    // 如果页面需要登录认证，但没有 token
    if (to.meta.requiresAuth && !token) {
        next('/login')  // 跳转到登录页
    } 
    // 如果已登录用户访问登录页，跳转到首页
    else if (to.path === '/login' && token) {
        next('/chat')
    }
    else {
        next()  // 正常访问
    }
})
```

#### 需要添加认证标记的页面

| 页面 | 路由 | 需要添加 `meta: { requiresAuth: true }` |
|------|------|---------------------------------------|
| 聊天 | `/chat` | ✅ |
| 知识库管理 | `/knowledge` | ✅ |
| 文档管理 | `/document` | ✅ |
| 登录 | `/login` | ❌ |
| 注册 | `/register` | ❌ |

## 5. 安全考虑

| 安全措施 | 说明 |
|---------|------|
| 密码加密 | 使用 bcrypt 加密，不可逆 |
| Token 签名 | 使用 HS256 算法签名 |
| Token 过期 | 7 天后需重新登录 |
| SQL 注入防护 | 使用 SQLAlchemy ORM |

## 6. 注意事项

1. **无需登出接口**：JWT 无状态，前端清除 Token 即可
2. **用户名校验**：需在后端做唯一性校验
3. **错误信息**：生产环境避免返回详细错误信息
