## 一、需求背景

### 1.1 问题描述

当前系统存在以下问题：

1. **页面刷新后历史丢失** - 对话记录仅存储在 Pinia Store 内存中，刷新页面后数据消失
2. **AI 无法记住上下文** - 后端 `ChatService._chat_histories` 是进程内内存，服务重启后丢失
3. **跨设备无法同步** - 用户切换设备后无法查看历史对话

### 1.2 预期目标

| 目标 | 描述 |
|------|------|
| 持久化存储 | 对话记录存入数据库，刷新页面不丢失 |
| 服务重启恢复 | 后端服务重启后可继续之前的对话上下文 |
| 用户数据隔离 | 每个用户只能查看自己的对话记录 |
| 统一历史管理 | 前端删除/切换会话时，后端同步删除/切换 |

### 3.2 数据模型设计

#### 3.2.1 新增表：会话表 `chat_sessions`

```python
class ChatSessionModel(Base):
    """会话模型"""
    __tablename__ = "chat_sessions"

    id = Column(String(36), primary_key=True)          # 会话 ID (UUID)
    user_id = Column(String(36), nullable=False, index=True)  # 用户 ID
    title = Column(String(200), default="新对话")        # 会话标题
    knowledge_base_id = Column(String(36), nullable=True)     # 关联知识库 (可选)
    created_at = Column(DateTime, default=datetime.now)
    updated_at = Column(DateTime, default=datetime.now, onupdate=datetime.now)
```

#### 3.2.2 新增表：消息表 `chat_messages`

```python
class ChatMessageModel(Base):
    """消息模型"""
    __tablename__ = "chat_messages"

    id = Column(String(36), primary_key=True)          # 消息 ID
    session_id = Column(String(36), ForeignKey("chat_sessions.id"), nullable=False, index=True)
    user_id = Column(String(36), nullable=False, index=True)     # 用户 ID (冗余存储，方便查询)
    role = Column(String(20), nullable=False)          # "user" 或 "assistant"
    content = Column(Text, nullable=False)             # 消息内容
    sources = Column(Text, nullable=True)              # AI 回复的引用来源 (JSON 字符串)
    created_at = Column(DateTime, default=datetime.now)
```

## 四、安全考虑

| 安全点 | 方案 |
|--------|------|
| 用户数据隔离 | 所有查询必须带上 `user_id` 条件 |
| 会话所有权验证 | 查询会话时验证 `session.user_id == current_user.id` |
| SQL 注入防护 | 使用 SQLAlchemy ORM 参数化查询 |
| 输入长度限制 | 消息内容最大 10000 字符 |

## 九、API 调用示例

### 9.1 创建新会话

```bash
curl -X POST "http://localhost:8000/api/v1/chat/sessions" \
  -H "Authorization: Bearer {token}" \
```
