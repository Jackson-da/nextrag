### 1.2 技术栈

| 层级 | 技术 | 说明 |
|------|------|------|
| 前端 | Vue3 + TypeScript | 组合式 API、类型安全 |
| UI | Element Plus | 企业级组件库 |
| 构建 | Vite | 快速热更新 |
| 后端 | FastAPI | 高性能、异步支持 |
| RAG | LangChain | 组件丰富、生态完善 |
| 向量库 | Chroma | 轻量级、易部署 |
| Embedding | BGE-large-zh | 中文优化 |
| LLM | DeepSeek-chat | 成本低、效果好 |
| 数据库 | SQLite | 零配置、嵌入式 |

## 3. 核心模块

### 3.1 文档处理流程

```
上传文件 → 格式验证 → 文档解析 → 智能分块 → 向量化 → 存储 → 返回结果
```

### 3.2 RAG 问答流程

```
用户问题 → Query处理 → 向量检索(Top-K) → 构建提示词 → 调用LLM → 返回答案+来源
```

### 4.1 SQLite 表结构

| 表名 | 用途 |
|------|------|

- 单一 collection，按 `kb_id` metadata 过滤
- 支持按知识库隔离检索

### 5.2 认证 API

#### 注册用户
```http
POST /api/v1/auth/register
Content-Type: application/json

Request:
{
  "username": "string (3-50字符)",
  "password": "string (6-100字符)"
}

Response (201):
{
  "message": "注册成功",
  "user_id": "uuid-string"
}
```

#### 用户登录
```http
POST /api/v1/auth/login
Content-Type: application/json

Request:
{
  "username": "string",
  "password": "string"
}

Response (200):
{
  "access_token": "jwt-token-string",
  "token_type": "bearer"
}
```

#### 获取当前用户
```http
GET /api/v1/auth/me
Authorization: Bearer {token}

Response (200):
{
  "id": "uuid-string",
  "username": "string"
}
```

### 5.3 会话管理 API

#### 创建会话
```http
POST /api/v1/chat/sessions
Authorization: Bearer {token}
Content-Type: application/json

Request (可选):
{
  "title": "string (最多200字符)",
  "knowledge_base_id": "uuid-string | null"
}

Response (201):
{
  "id": "uuid-string",
  "user_id": "uuid-string",
  "title": "新对话",
  "knowledge_base_id": null,
  "message_count": 0,
  "created_at": "2026-04-29T10:00:00",
```

## 7. 会话管理架构

### 7.1 会话数据模型

```
┌─────────────────┐       ┌─────────────────┐
│  chat_sessions  │       │ chat_messages   │
├─────────────────┤       ├─────────────────┤
│ id (PK)         │──┐    │ id (PK)         │
│ user_id (FK)    │  │    │ session_id (FK) │──┐
│ title           │  └───►│ user_id         │  │
│ kb_id           │       │ role            │  │
│ created_at      │       │ content         │  │
│ updated_at      │       │ sources         │  │
└─────────────────┘       │ created_at      │  │
        │                  └─────────────────┘  │
        │                                  │
        ▼                                  ▼
┌─────────────────────────────────────────────────┐
│                    users                        │
├─────────────────────────────────────────────────┤
│ id (PK)                                         │
│ username                                        │
│ password_hash                                   │
└─────────────────────────────────────────────────┘
```

### 7.2 会话管理流程

```
┌─────────────────────────────────────────────────────────────────┐
│                     前端初始化                                    │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│ 1. 加载会话列表: GET /api/v1/chat/sessions                       │
│ 2. 渲染侧边栏会话列表                                            │
│ 3. 加载最新会话消息: GET /api/v1/chat/sessions/{id}/messages     │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│                     用户发送消息                                  │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│ 1. 发送消息: POST /api/v1/chat/stream                            │
│ 2. 后端保存用户消息到 chat_messages                              │
│ 3. 流式返回 AI 回复                                             │
│ 4. 后端保存 AI 回复到 chat_messages                              │
│ 5. 更新会话 updated_at                                           │
└─────────────────────────────────────────────────────────────────┘
```
