```

├─────────────────────────────────────────────────────────────────┤
│  ┌─────────────────┐                                             │
│  │  前端层 (Vue3)  │  文档上传页 │ 问答对话页 │ 知识库页 │ 管理后台 │
│  └────────┬────────┘                                             │
│           │ REST API                                              │
│           ▼                                                       │
│  ┌─────────────────┐                                             │
│  │ API层 (FastAPI) │  文档管理 │ 问答 │ 知识库 │ 会话管理 │ 系统管理 │
│  └────────┬────────┘                                             │
│           │                                                      │
│           ▼                                                       │
│  ┌─────────────────┐                                             │
│  │   业务服务层     │  DocumentService │ ChatService │ KnowledgeSvc │
│  └────────┬────────┘                                             │
│           │                                                      │
│           ▼                                                       │
│  ┌─────────────────┐                                             │
│  │ LangChain 核心层 │  文档加载 │ 分割 │ Embeddings │ RAG链      │
│  └────────┬────────┘                                             │
│           │                                                      │
│           ▼                                                       │
│  ┌─────────────────┐                                             │
│  │     数据层       │  SQLite(元数据) │ Chroma(向量) │ Redis(缓存) │
```

### 1.2 技术栈

| 层级 | 技术 | 说明 |
|------|------|------|
| 前端 | Vue3 + TypeScript | 组合式 API、类型安全 |
| UI | Element Plus | 企业级组件库 |
| 构建 | Vite | 快速热更新 |
| 后端 | FastAPI | 高性能、异步支持 |
| RAG | LangChain | 组件丰富、生态完善 |
| 向量库 | Chroma | 轻量级、易部署 |
| Embedding | BGE-large-zh | 中文优化 |
| LLM | DeepSeek-chat | 成本低、效果好 |
| 数据库 | SQLite | 零配置、嵌入式 |

## 3. 核心模块

### 3.1 文档处理流程

```
上传文件 → 格式验证 → 文档解析 → 智能分块 → 向量化 → 存储 → 返回结果
```

### 3.2 RAG 问答流程

```
用户问题 → Query处理 → 向量检索(Top-K) → 构建提示词 → 调用LLM → 返回答案+来源
```

## 4. 数据库设计

### 4.1 SQLite 表结构

| 表名 | 用途 |
|------|------|
| documents | 文档元数据 |
| document_chunks | 文档分块 |
| knowledge_bases | 知识库 |
| users | 用户信息 |
| chat_sessions | 聊天会话 |
| chat_messages | 聊天消息 |

### 4.2 Chroma 向量库

- 单一 collection，按 `kb_id` metadata 过滤
- 支持按知识库隔离检索

### 7.2 会话管理流程

```
┌─────────────────────────────────────────────────────────────────┐
│                     前端初始化                                    │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│ 1. 加载会话列表: GET /api/v1/chat/sessions                       │
│ 2. 渲染侧边栏会话列表                                            │
│ 3. 加载最新会话消息: GET /api/v1/chat/sessions/{id}/messages     │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│                     用户发送消息                                  │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│ 1. 发送消息: POST /api/v1/chat/stream                            │
│ 2. 后端保存用户消息到 chat_messages                              │
│ 3. 流式返回 AI 回复                                             │
│ 4. 后端保存 AI 回复到 chat_messages                              │
│ 5. 更新会话 updated_at                                           │
└─────────────────────────────────────────────────────────────────┘
```
