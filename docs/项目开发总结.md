## 一、项目概述

### 1.1 项目简介

**NextRAG** 是一个基于 RAG (Retrieval-Augmented Generation) 的智能文档问答系统，支持文档上传、向量检索和智能问答功能。

### 1.2 核心特性

| 特性 | 说明 |
|------|------|
| 📄 文档管理 | 支持 PDF、TXT、Markdown、DOCX 格式上传 |
| 🔍 智能检索 | 基于向量数据库的语义检索 |
| 💬 多轮对话 | 支持上下文记忆的多轮对话 |
| ⚡ 流式输出 | 实时流式响应（SSE） |
| 🏢 企业级 | 统一日志、结构化输出、微服务架构 |

### 1.3 技术栈

| 层级 | 技术 |
|------|------|
| **前端** | Vue 3 + TypeScript + Vite + Pinia + Axios |
| **后端** | FastAPI + Python 3.10+ + uvicorn |
| **LLM** | DeepSeek API (OpenAI 兼容) |
| **Embedding** | BGE-large-zh-v1.5 |
| **向量库** | ChromaDB |
| **数据库** | SQLite (aiosqlite) |
| **文档处理** | PyPDF, python-docx |

## 五、API 接口

### 5.1 对话 API (`/api/v1/chat`)

| 接口 | 方法 | 描述 |
|------|------|------|
| `/chat` | POST | 非流式问答，返回完整回答 |
| `/stream` | POST | 流式问答（Server-Sent Events） |
| `/history/{session_id}` | GET | 获取对话历史 |
| `/history/{session_id}` | DELETE | 清除对话历史 |
| `/health` | POST | 问答服务健康检查 |

### 5.2 文档 API (`/api/v1/documents`)

| 接口 | 方法 | 描述 |
|------|------|------|
| `/upload` | POST | 上传文档（支持 PDF/TXT/MD/DOCX） |
| `/` | GET | 获取文档列表（支持分页、知识库过滤） |
| `/{document_id}` | GET | 获取文档详情 |
| `/{document_id}` | DELETE | 删除文档（同时删除向量） |
| `/vectorstore/info` | GET | 获取向量库信息（文档数等） |

### 5.3 知识库 API (`/api/v1/knowledge-bases`)

| 接口 | 方法 | 描述 |
|------|------|------|
| `/` | POST | 创建知识库 |
| `/` | GET | 获取知识库列表 |
| `/{kb_id}` | GET | 获取知识库详情 |
| `/{kb_id}` | PUT | 更新知识库 |
| `/{kb_id}` | DELETE | 删除知识库（同时删除关联文档） |
| `/{kb_id}/documents` | GET | 获取知识库关联的文档列表 |

### 5.4 系统 API

| 接口 | 方法 | 描述 |
|------|------|------|
| `/api/v1/ping` | GET | 存活探测（用于负载均衡） |
| `/api/v1/health` | GET | 健康检查（返回 LLM 和向量库状态） |
| `/health` | GET | 极简健康检查（无前缀，用于容器探测） |
| `/` | GET | 首页（返回服务信息和文档链接） |

### 5.5 响应模型

```typescript
// 问答响应

  answer: string;        // 回答内容

// 文档列表响应

  items: Document[];     // 文档列表

}
```

## 十四、知识库选择功能 (v1.1)

### 14.1 功能说明

支持在问答时选择特定知识库，实现文档的隔离检索：

```
┌─────────────────────────────────────────────────────────┐
│                    知识库选择流程                         │
├─────────────────────────────────────────────────────────┤
│                                                          │
│   用户选择知识库 KB1 ──▶ 提问                            │
│          │                    │                          │
│          ▼                    ▼                          │
│   kb_id=KB1 ───────────▶ 检索过滤                       │
│                              │                          │
│                              ▼                          │
│                    只返回 KB1 的文档                      │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

### 14.2 技术实现

**后端实现**：

| 文件 | 修改内容 |
|------|----------|
| `rag_chain.py` | 接收 kb_id，构建带过滤的检索器 |
| `document_service.py` | 上传时将 kb_id 添加到向量库元数据 |
| `chat_service.py` | 按 kb_id 缓存 RAGChainBuilder 实例 |

**关键代码**：

```python
# 检索器按 kb_id 过滤
search_kwargs = {"k": 5}
if kb_id:
    search_kwargs["filter"] = {"kb_id": kb_id}

# 文档上传时添加 kb_id 元数据
chunk.metadata["kb_id"] = knowledge_base_id
```

**前端实现**：

| 文件 | 功能 |
|------|------|
| `ChatView.vue` | 知识库选择下拉框 UI |
| `chat.ts` | 知识库状态管理和 API 调用 |

### 14.3 使用方式

1. **选择知识库问答**：选择指定知识库，只检索该知识库的文档
2. **全局检索**：不选择知识库，检索所有文档
3. **切换知识库**：切换后自动使用新的检索范围

### 14.4 注意事项

- 旧文档（v1.0 上传）没有 `kb_id` 元数据，需要重新上传
- 不同知识库使用同一向量库集合，通过元数据过滤实现隔离
- RAG 链按 `kb_id` 缓存，避免重复创建
