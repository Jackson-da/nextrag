## 一、需求背景

当前项目使用 `structlog` 进行日志记录，需要实现持久化和统一格式。

### 现状分析

| 问题 | 说明 | 状态 |
|------|------|------|
| 仅控制台输出 | 服务关闭后日志丢失 | ✅ 已解决 |
| 无请求追踪 | 缺少 `trace_id` 贯穿请求链路 | ✅ 已解决 |
| 无分级配置 | 无法动态调整日志级别 | ✅ 已解决 |
| 部分 API 缺少日志 | `document.py`、`knowledge.py` 未使用日志 | ✅ 已解决 |
| 日志格式不统一 | 每条日志字段不一致 | ✅ 已解决 |

## 五、详细设计

### 5.1 日志格式标准

#### 统一结构（每条日志固定字段）

```json
{
  "timestamp": "2026-04-27T16:00:00.000000+08:00",
  "level": "INFO",
  "trace_id": "a1b2c3d4",
  "service": "smart-qa-system",
  "env": "development",
  "logger": "app.api.document",
  "msg": "获取文档列表",
  "extra": {
    "total": 3,
    "skip": 0,
    "limit": 20
  }
}
```

#### 字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| timestamp | string | ✅ | ISO 8601 格式，含毫秒和时区 |
| level | string | ✅ | DEBUG/INFO/WARN/ERROR |
| trace_id | string | ✅ | 8位追踪ID，从请求头或自动生成 |
| service | string | ✅ | 服务名称 |
| env | string | ✅ | 环境标识 |
| logger | string | ✅ | 日志来源（模块路径） |
| msg | string | ✅ | 日志消息 |
| extra | object | ❌ | 业务扩展数据 |

### 5.2 core/logging.py 实现

**核心功能：**
- 环境适配（开发/生产）
- 结构化 JSON 输出
- 固定字段结构
- 日志轮转（10MB，保留7天）

```python
# 关键代码片段

def setup_logging():
    """初始化日志配置"""
    settings = get_settings()
    
    # 确定环境
    env = settings.environment
    
    # 共享 processors
    shared_processors = [
        structlog.contextvars.merge_contextvars,
        structlog.stdlib.filter_by_level,
        structlog.stdlib.add_logger_name,
        structlog.stdlib.add_log_level,
        structlog.stdlib.PositionalArgumentsFormatter(),
        structlog.processors.TimeStamper(fmt="iso"),
        structlog.processors.StackInfoRenderer(),
        structlog.processors.format_exc_info,
        structlog.processors.UnicodeDecoder(),
        add_service_info,          # 添加 service/env
        add_trace_id,              # 添加 trace_id
        standardize_fields,        # 统一字段结构
    ]
    
    # 开发环境：彩色控制台 + JSON 文件
    if env == "development":
        processors = shared_processors + [
            structlog.dev.ConsoleRenderer(colors=True),
        ]
    # 生产环境：纯 JSON 输出
    else:
        processors = shared_processors + [
            structlog.processors.JSONRenderer(),
        ]
    
    structlog.configure(
        processors=processors,
        context_class=dict,
        logger_factory=structlog.stdlib.LoggerFactory(),
        wrapper_class=structlog.stdlib.BoundLogger,
        cache_logger_on_first_use=True,
    )
```

### 5.3 middleware/logging.py 实现

**核心功能：**
- 自动生成 trace_id
- 支持请求头传递（`X-Trace-ID`）
- 记录请求开始/完成
- 计算请求耗时
- 绑定上下文变量

```python
# 关键代码片段

class LoggingMiddleware(BaseHTTPMiddleware):
    async def dispatch(self, request: Request, call_next):
        # 1. 获取或生成 trace_id
        trace_id = request.headers.get("x-trace-id") or str(uuid4())[:8]
        request.state.trace_id = trace_id
        
        # 2. 绑定上下文（所有日志都会带上）
        structlog.contextvars.bind_contextvars(trace_id=trace_id)
        
        # 3. 记录请求开始
        log = get_logger()
        start_time = time.time()
        log.info("请求开始", method=request.method, path=str(request.url.path))
        
        # 4. 处理请求...
        
        # 5. 记录请求完成
        log.info("请求完成", status_code=response.status_code, duration=duration)
```

### 5.4 使用方式

```python
# 1. 获取日志器
log = get_logger()

# 2. 普通日志
log.info("消息内容")
log.warning("警告信息")
log.error("错误信息", error="详细错误")

# 3. 带业务数据
log.info("获取文档列表", extra={"total": 10, "skip": 0, "limit": 20})

# 4. 带 trace_id（自动从上下文获取）
log = get_logger(trace_id="自定义ID")
log.info("用户登录", extra={"user_id": "123"})

# 5. 使用 LogContext
with LogContext(user_id="123", action="login"):
    log.info("用户操作")  # 自动带上 user_id 和 action
```
