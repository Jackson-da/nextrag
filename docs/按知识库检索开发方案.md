## 一、需求分析

### 1.1 现状问题

| 问题 | 说明 |
|------|------|
| 全局检索 | 当前问答检索所有知识库的文档，无法指定特定知识库 |
| 参数未使用 | `ChatRequest.knowledge_base_id` 已定义但未实现 |
| 检索不精准 | 返回内容可能来自无关知识库 |

### 1.2 目标

```
用户提问 + 指定知识库
       ↓
只检索该知识库下的文档
       ↓
返回精准答案
```

### 1.3 方案选型

| 方案 | 描述 | 优点 | 缺点 |
|------|------|------|------|
| **方案A：Metadata 过滤** | 单一 collection，通过 filter 参数按 kb_id 过滤 | 简单、改动小 | 数据量大时性能可能下降 |
| **方案B：多 Collection** | 每个知识库一个 ChromaDB collection | 性能好、数据隔离 | 改动大、需要迁移数据 |

**推荐方案：A（Metadata 过滤）**

理由：
1. 改动最小，复用现有架构
2. ChromaDB 对 metadata 过滤支持良好
3. 适合中小规模数据（<100万向量）

### 2.3 检索逻辑

```python
def get_retriever(kb_id: str | None) -> VectorStoreRetriever:
    """
    获取检索器
    
    Args:
        kb_id: 知识库 ID，None 表示全局检索
        
    Returns:
        配置好的检索器
    """
    if kb_id:
        # 按知识库过滤
        return vectorstore.as_retriever(
            search_kwargs={
                "k": top_k,
                "filter": {"kb_id": kb_id}
            }
        )
    else:
        # 全局检索（不过滤）
        return vectorstore.as_retriever(
            search_kwargs={"k": top_k}
        )
```

## 五、详细设计

### 5.1 VectorStoreManager 修改

```python
# core/vectorstore.py

def get_retriever(self, kb_id: str | None = None) -> VectorStoreRetriever:
    """
    获取带知识库过滤的检索器
    
    Args:
        kb_id: 知识库 ID，None 表示全局检索
        
    Returns:
        配置好的检索器
    """
    settings = get_settings()
    search_kwargs = {"k": settings.retrieval_top_k}
    
    # 如果指定了知识库，添加过滤条件
    if kb_id:
        search_kwargs["filter"] = {"kb_id": kb_id}
    
    return self.vectorstore.as_retriever(
        search_kwargs=search_kwargs
    )
```

### 5.2 RAGChainBuilder 修改

```python
# core/rag_chain.py

class RAGChainBuilder:
    def __init__(
        self,
        llm,
        retriever=None,
        kb_id: str | None = None,  # 新增参数
    ):
        # ...
        self.kb_id = kb_id
    
    def _get_retriever(self):
        """获取检索器，支持知识库过滤"""
        if self.retriever:
            # 使用外部传入的检索器
            return self.retriever
        else:
            # 创建带 kb_id 过滤的检索器
            vectorstore = VectorStoreManager(embedding=self.embedding)
            return vectorstore.get_retriever(kb_id=self.kb_id)
```

### A. ChromaDB filter 语法

```python
# 基础过滤
filter = {"kb_id": "kb-123"}

# 多个条件
filter = {"kb_id": {"$in": ["kb-1", "kb-2"]}}  # OR

filter = {"$and": [{"kb_id": "kb-1"}, {"source": {"$contains": "pdf"}}]}  # AND
```
